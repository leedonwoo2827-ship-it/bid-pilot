# /brief

오늘의 조달 현황을 브리핑 형태로 정리한다.

## 실행 흐름

### Step 1: 컨텍스트 로딩 (context-loader)

`_context/` 폴더에서 회사 데이터를 로드하여 적합도 평가에 활용한다.

### Step 2: 공고 스캔 (procurement-monitor)

주요 조달 채널에서 최근 공고를 웹 검색으로 수집한다.

스캔 대상 (사용자의 사업 영역에 맞게 조정):
- 나라장터 (g2b.go.kr) — 국내 공공조달
- KOICA (koica.go.kr) — ODA 사업
- NIPA (nipa.kr) — ICT 분야
- NIA (nia.or.kr) — 디지털 전환
- ADB (adb.org) — 아시아개발은행
- 기타 사용자 지정 채널

사용자에게 관심 분야/키워드를 확인한다:
- "어떤 분야의 공고를 찾으시겠습니까? (예: 교육, ICT, 컨설팅, ODA)"
- "특정 발주처가 있으신가요? (예: KOICA, 나라장터)"

### Step 3: 적합도 평가

각 공고에 대해 회사 데이터를 기반으로 자동 적합도 평가:
- 분야 적합성 (회사 역량과 매칭)
- 자격 충족도 (실적, 매출, 인력)
- 예산/일정 적정성

### Step 4: 브리핑 문서 생성

```
# 수주 일일 브리핑 — [날짜]

## 신규 공고
| 순위 | 사업명 | 발주처 | 마감일 | 예산규모 | 적합도 |
|------|--------|--------|--------|----------|--------|
| 1 | [사업명] | [기관] | [날짜] (D-XX) | [금액] | 🟢 |
| 2 | [사업명] | [기관] | [날짜] (D-XX) | [금액] | 🟡 |
| 3 | [사업명] | [기관] | [날짜] (D-XX) | [금액] | 🔴 |

## 마감 임박 (7일 이내)
| 사업명 | 발주처 | 마감일 | 상태 |
|--------|--------|--------|------|
| [사업명] | [기관] | [날짜] (D-3) | 제안서 작성 중 |

## 시장 동향
- [이번 주 주요 변화 1]
- [이번 주 주요 변화 2]

## 추천 액션
- [사업명] → /bid-analyze 권장 (적합도 높음, D-21)
- [사업명] → 추가 정보 확인 필요 (자격 확인 중)
- [사업명] → 모니터링 지속 (공고 예정)
```

### Step 5: 저장 및 후속 안내

- 브리핑 문서를 `_analysis/brief-[날짜].md`로 저장
- 사용자에게 후속 액션 안내:
  - "관심 공고가 있으시면 번호를 말씀해주세요. RFP 분석을 시작합니다."
  - "HWP/PPT로 변환하시겠습니까?"

## 자동 스케줄링

Claude Code의 `/schedule` 기능과 연동하여 자동 실행 가능:

```
매 평일 오전 8시에 /brief 자동 실행
→ _analysis/brief-[날짜].md 자동 생성
```

설정 방법은 INSTALL.md를 참조한다.

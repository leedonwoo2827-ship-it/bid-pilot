# /bid-analyze

RFP를 받았을 때 수주 전략 수립에 필요한 전체 분석을 순서대로 실행하는 워크플로우.

## 실행 흐름

### Step 0: 컨텍스트 로딩 (context-loader)

`_context/` 폴더가 있으면 먼저 회사 데이터를 로드한다.
- 회사 프로필, 수행실적, 인력 풀 구축
- 이후 모든 단계에서 회사 데이터를 자동 반영

### Step 1: RFP 분석 (analyze-rfp)

RFP/TOR 문서를 체계적으로 분석한다.

- 파일 첨부 → 파일을 읽고 분석
- 텍스트 붙여넣기 → 해당 내용 분석
- 사업명만 제공 → 알려진 정보 기반 분석 + 추가 정보 요청

**산출물:**
- 사업 개요 정리
- 평가기준 배점표 구조화
- 자격요건 체크리스트 (회사 데이터와 자동 대조)
- 투입인력 요구사항
- 리스크 초기 식별

### Step 2: 경쟁사 매핑 (competitor-mapping)

RFP 분석 결과를 기반으로 예상 경쟁사를 분석한다.

- 사용자에게 알고 있는 경쟁사 정보를 물어본다
- 해당 분야의 주요 플레이어를 식별한다
- 정량 비교 매트릭스를 작성한다

**산출물:**
- 예상 경쟁사 3~5개사 프로파일
- 정량 비교 매트릭스
- 포지셔닝 분석

### Step 3: 컨소시엄 전략 (consortium-strategy)

경쟁 분석 결과를 바탕으로 최적의 컨소시엄 구성안을 설계한다.

- 컨소시엄 필요성 진단 (단독 vs 컨소시엄)
- 필요 역할과 파트너 유형 정의
- 1~2개의 컨소시엄 구성안 제시

**산출물:**
- 컨소시엄 구성 권고안
- 역할 배분 매트릭스
- 지분율 제안

### Step 4: 자가평가 (bid-scoring)

최종적으로 현재 상태에서의 수주 가능성을 종합 평가한다.

- RFP 배점표 기반 항목별 채점 (회사 데이터 근거 연결)
- 강점/약점 종합
- Gap 분석
- Go/No-Go 권고

**산출물:**
- 항목별 자가 점수표 (근거 포함)
- Gap Closure Plan
- Go/No-Go 판단

## 진행 원칙

- 각 단계가 완료될 때마다 결과를 제시하고, 사용자의 피드백을 반영한 후 다음 단계로 진행
- 사용자가 특정 단계를 건너뛰고 싶다면 유연하게 대응
- 정보가 부족한 부분은 가정(assumption)을 명시하고 확인 요청
- 최종 결과는 하나의 종합 보고서로 통합

## 최종 산출물 구조

```
# 수주 분석 종합 보고서: [사업명]

## Executive Summary
[1페이지 요약: 사업 개요, 핵심 판단, Go/No-Go 권고]

## Part 1: RFP 분석
[analyze-rfp 결과 — 자격요건 대조 매트릭스 포함]

## Part 2: 경쟁 환경 분석
[competitor-mapping 결과 — 정량 비교 매트릭스 포함]

## Part 3: 컨소시엄 전략
[consortium-strategy 결과]

## Part 4: 수주 가능성 평가
[bid-scoring 결과 — Gap 분석 포함]

## 다음 단계 액션 플랜
1. [즉시] ...
2. [1주 내] ...
3. [제안서 작성 시] ...
```

- 문서 변환이 필요하면: "보고서를 HWP/PPT로 변환하시겠습니까?" → export-deliverables

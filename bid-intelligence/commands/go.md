# /go

수주 분석 도우미의 진입점. 사용자가 무엇을 해야 할지 모를 때 여기서 시작한다.

## 실행 흐름

사용자에게 다음 메뉴를 한국어로 제시한다:

```
수주 도우미입니다. 무엇을 하시겠습니까?

1. RFP 분석 — 입찰공고/RFP 파일을 첨부하세요
2. 공고 스캔 — 최근 조달공고를 분석합니다
3. 경쟁 분석 — 특정 사업의 경쟁 환경을 분석합니다
4. 제안서 작성 — 분석 결과를 기반으로 제안서 초안을 작성합니다
5. 문서 변환 — 마크다운을 HWP 또는 PPT로 변환합니다
6. 일일 브리핑 — 오늘의 조달 현황을 확인합니다

번호를 선택하거나, 파일을 첨부해주세요.
```

## 라우팅 규칙

### 번호 선택 시

| 선택 | 실행 |
|------|------|
| 1 | `/bid-analyze` 실행. 파일 첨부를 요청 |
| 2 | `/scan` 실행. 관심 분야/발주처를 질문 |
| 3 | `/compete` 실행. 사업명/분야를 질문 |
| 4 | proposal-writing 스킬 안내. 분석 결과가 있는지 확인 |
| 5 | export-deliverables 스킬 안내. 변환할 파일을 요청 |
| 6 | `/brief` 실행 |

### 파일 첨부 시 (자동 감지)

첨부된 파일의 내용을 빠르게 확인하여 유형을 판별한다:

- **RFP/TOR/입찰공고** (제안요청서, 과업지시서, 입찰 공고 키워드 감지) → "RFP로 판단됩니다. 전체 수주 분석을 시작할까요?" → `/bid-analyze`
- **조달공고/입찰공고 목록** → "조달공고입니다. 적합도를 평가할까요?" → `/scan`
- **제안서/보고서** → "기존 제안서로 보입니다. _context/proposals/에 저장하시겠습니까?"
- **판별 불가** → "이 문서의 용도를 알려주세요" (선택지 제시)

### 텍스트 입력 시

사용자가 번호 대신 자연어로 입력하면 의도를 파악하여 적절한 워크플로우로 라우팅:

- "이 공고 분석해줘" → `/bid-analyze`
- "경쟁사가 궁금해" → `/compete`
- "오늘 새 공고 있어?" → `/brief`
- "제안서 써야 해" → proposal-writing 안내

## 초기 설정 안내

_context/ 폴더가 비어있거나 회사 데이터가 부족하면, 분석 전에 다음을 안내:

```
회사 데이터가 부족합니다. 분석 품질을 높이려면 _context/ 폴더에 다음 자료를 넣어주세요:

필수:
- IR보고서 또는 회사소개서 (PDF/MD)
- 과거 제안서 1건 이상 (PDF/MD)

권장:
- 최근 3개년 재무제표
- 핵심 인력 CV
- 부서별 전문 자료

자료를 추가한 후 다시 /go를 실행하세요.
```

## 진행 원칙

- 항상 한국어로 응답한다
- 전문 용어는 최소화하고 쉬운 말로 안내한다
- 각 단계에서 사용자가 다음에 할 일을 명확히 알려준다
- 사용자가 길을 잃으면 언제든 `/go`로 돌아올 수 있음을 안내한다

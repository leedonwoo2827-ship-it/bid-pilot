# 컨텍스트 로더 (Context Loader)

프로젝트의 `_context/` 폴더에 있는 회사 데이터를 자동으로 읽고 구조화하여, 모든 분석 스킬에 회사 정보를 제공한다.

## 왜 이 스킬이 필요한가

입찰 분석에서 "우리 회사"의 역량 데이터가 없으면 일반론만 나온다. RFP 자격요건을 대조하려면 실적이 필요하고, 채점하려면 재무/인력 데이터가 필요하다. 이 스킬은 _context/ 폴더에 넣어둔 원천 데이터를 읽어 모든 분석에 회사 맥락을 자동으로 주입한다.

## 실행 절차

### 1단계: 폴더 스캔

프로젝트 루트에서 `_context/` 폴더를 찾는다.

- **폴더가 있으면**: 하위의 모든 파일을 재귀적으로 탐색
- **폴더가 없으면**: 사용자에게 안내 — "회사 데이터가 없습니다. _context/ 폴더에 IR보고서, 과거 제안서 등을 넣어주세요."

지원 파일 형식:
- **마크다운 (.md)**: 직접 읽기
- **PDF (.pdf)**: 텍스트 추출하여 분석
- **텍스트 (.txt)**: 직접 읽기
- **SRT (.srt)**: 자막 텍스트 추출
- 기타 형식은 파일명과 경로만 기록

### 2단계: 정보 추출

파일 내용에서 다음 카테고리의 정보를 추출한다. 어떤 폴더 구조든, 어떤 파일 형식이든 내용 기반으로 판단한다:

**A. 회사 개요**
- 회사명, 설립일, 대표자
- 사업 분야, 주소
- 업력

**B. 재무 현황**
- 매출액 (연도별)
- 자본금
- 신용등급
- 사업부문별 매출 구성

**C. 조직 구성**
- 부서별 인원
- 총 인원
- 조직도 (있는 경우)

**D. 수행실적**
각 실적에 대해 다음을 추출:
- 사업명
- 발주처 (KOICA, ADB, 나라장터, 민간 등)
- 수행 기간
- 사업 금액
- 분야 (교육, ICT, 컨설팅 등)
- 국가/지역
- 역할 (주관사, 공동참여 등)

**E. 핵심 역량**
- 보유 기술/솔루션 (LMS, 플랫폼, AI 등)
- 인증/자격/특허
- 수상 이력
- 차별화 요소

**F. 인력 풀**
- 핵심 전문가 이력 (있는 경우)
- 분야별 전문가 수
- PM급 인력 경력

### 3단계: 검증 리포트 생성

추출한 정보의 완성도를 검증하고 리포트를 출력한다:

```
## 컨텍스트 로딩 결과

### 데이터 소스
- 읽은 파일: [파일 수]개
  - [파일명1] (유형: IR보고서)
  - [파일명2] (유형: 과거 제안서)
  - ...

### 확인된 정보
| 카테고리 | 항목 | 데이터 | 출처 파일 |
|----------|------|--------|-----------|
| 회사 개요 | 회사명 | (주)OOO | file1.md |
| 재무 | 최근 매출 | XX억원 (20XX) | file2.pdf |
| 실적 | 유사실적 수 | XX건 | file1.md |
| ... | ... | ... | ... |

### 보완 필요 항목
- ⚠️ [항목]: [이유] (예: 최신 재무 데이터 없음, 가장 최근 2021년)
- ⚠️ [항목]: [이유]

### 권장 추가 데이터
- [추가하면 분석 품질이 올라가는 데이터 목록]
```

### 4단계: 프로필 요약 생성

추출한 데이터를 하나의 구조화된 프로필로 정리한다. 이 프로필이 다른 스킬에서 참조하는 기준 데이터가 된다.

## 다른 스킬에서의 호출 방식

다른 스킬(analyze-rfp, bid-scoring 등)이 실행될 때:

1. `_context/` 폴더가 존재하는지 확인
2. 존재하면 context-loader를 먼저 실행하여 회사 프로필 구축
3. 프로필 데이터를 분석에 반영
4. 존재하지 않으면 사용자에게 회사 정보를 직접 질문

## 데이터 최신성 주의사항

- 추출한 데이터에 **연도를 반드시 표기**하여, 최신 여부를 판단할 수 있게 한다
- 3년 이상 된 재무 데이터는 "⚠️ 오래된 데이터"로 표시
- 인원/조직 데이터는 2년 이상이면 "⚠️ 확인 필요"로 표시
- 사용자가 데이터를 추가/수정한 후 다시 실행하면 갱신된 프로필을 생성

## 권장 폴더 구조

폴더 구조를 강제하지 않지만, 다음 구조를 안내한다:

```
_context/
  company/       # 회사 소개 자료, IR보고서, 신용평가
  proposals/     # 과거 제안서 (MD/PDF)
  credentials/   # 인증서, 수상 이력
  personnel/     # 핵심 인력 CV
  sector/        # 부서별 전문 데이터
                 #   예) education/: 강의 SRT, 커리큘럼.md
                 #   예) it/: 기술명세서, 아키텍처문서
                 #   예) overseas/: 국가별 조사자료
```

이 구조를 따르면 정보 추출의 정확도가 높아지지만, 모든 파일을 _context/ 루트에 넣어도 동작한다.

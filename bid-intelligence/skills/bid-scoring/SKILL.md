# 입찰 자가평가 (Bid Scoring)

RFP 배점표를 기준으로 제안서(또는 제안서 계획)를 자가 채점하여, 수주 가능성을 진단하고 개선 포인트를 도출한다.

## 왜 자가평가가 필요한가

제안서를 쓰다 보면 자기 관점에 갇히기 쉽다. 평가위원은 배점표의 각 항목에 기계적으로 점수를 매긴다. 제안서 제출 전에 평가위원의 시선으로 한 번 점검하면, 치명적인 약점을 발견하고 보완할 시간을 확보할 수 있다.

## 컨텍스트 활용

이 스킬을 실행하기 전에, 프로젝트의 `_context/` 폴더가 있는지 확인한다.

**폴더가 있으면:**
1. context-loader로 회사 프로필 구축
2. 채점 시 **구체적 근거를 인용** — "수행실적 점수: 85/100 — 근거: KOICA 교육 사업 3건, ADB 컨설팅 2건 (_context/proposals/에서 확인)"
3. Gap 분석에 회사 데이터 기반 개선 방안 제시

**폴더가 없으면:**
- 사용자에게 필요 정보를 질문하여 채점
- 근거 없는 항목은 보수적으로 채점

## 평가 절차

### 1단계: 배점표 구조화

RFP의 평가기준 배점표를 정확히 구조화한다.

- 대항목 / 중항목 / 소항목 구분
- 각 항목의 배점
- 평가 등급별 점수 범위 (우수/양호/보통/미흡 등)
- 정량 평가 항목과 정성 평가 항목 구분

### 2단계: 항목별 자가 채점

각 평가항목에 대해 5단계로 채점한다:

| 등급 | 점수 비율 | 기준 |
|---|---|---|
| 우수 | 90~100% | 평가위원이 감탄할 수준. 차별화된 내용, 구체적 근거 |
| 양호 | 75~89% | 요구사항을 충실히 충족. 추가적 강점 있음 |
| 보통 | 60~74% | 기본 요구사항은 충족. 차별화 요소 부족 |
| 미흡 | 40~59% | 일부 요구사항 누락 또는 불충분 |
| 부적합 | 0~39% | 핵심 요구사항 미충족. 치명적 약점 |

### 3단계: 근거 연결 채점 (컨텍스트 연동)

_context/ 데이터를 활용하여 각 채점에 구체적 근거를 연결한다:

```
### 사업수행능력 (배점: 30점)

- 자가 점수: 25/30점 (83%)
- 등급: 양호
- 근거:
  - 유사실적 7건 확인 (KOICA 3건, ADB 2건, NIPA 2건)
  - 실적 금액 합계: 약 120억원
  - 해당 국가 실적: 2건 (베트남, 우즈베키스탄)
  - 출처: _context/proposals/나이지리아-제안서.md
- 강점: 교육/ICT 분야 실적 풍부
- 약점: 해당 국가(수원국) 직접 실적 부족
- 개선방안: 컨소시엄 파트너로 현지 실적 보완
```

### 4단계: Gap 분석

점수가 낮은 항목별로 구체적인 Gap Closure Plan을 작성한다:

```
## Gap 분석

| 항목 | 현재 점수 | 목표 점수 | Gap | 개선 방안 | 실행 난이도 |
|------|-----------|-----------|-----|-----------|-------------|
| 현지 네트워크 | 12/20 | 17/20 | 5점 | 현지 파트너 컨소시엄 | 중 |
| PM 경력 | 7/10 | 9/10 | 2점 | OO 이사 PM 배정 | 하 |
| ... | ... | ... | ... | ... | ... |

### 우선순위 (배점 × Gap = 영향도)
1. [가장 영향도 높은 항목] — 즉시 조치
2. [두 번째] — 1주 내 해결
3. [세 번째] — 제안서 작성 시 반영
```

### 5단계: 종합 점수 및 수주 전망

- 항목별 점수를 합산하여 예상 총점 산출
- 경쟁사 예상 점수와 비교 (competitor-mapping 결과가 있는 경우)
- 수주 가능성을 3단계로 평가: 유리 / 경합 / 불리
- Go/No-Go 최종 판단 근거 제시

## 출력 형식

```
# 입찰 자가평가: [사업명]

## 1. 배점표 구조
[항목, 배점, 비율]

## 2. 항목별 채점
[항목별 점수 + 근거 + 강점/약점/개선방안]

## 3. 종합 점수표
| 항목 | 배점 | 자가점수 | 비율 | 등급 | 근거 요약 |
|---|---|---|---|---|---|
| ... | ... | ... | ...% | ... | ... |
| **합계** | **XX** | **XX** | **XX%** | | |

## 4. Gap 분석
[Gap Closure Plan 매트릭스]

## 5. 수주 전망
- 예상 총점: XX/100

[종합 판단 및 근거]
```

## 채점 시 유의사항

- 자기 평가는 본질적으로 관대해지기 쉽다. 의도적으로 보수적으로 채점하는 것이 현실적인 예측에 도움이 된다
- 정량 항목(실적 건수, 매출액 등)은 증빙 가능한 것만 반영한다
- 평가위원은 제안서에 명시적으로 쓰여 있는 것만 평가한다
- 범분야 이슈(젠더, 환경)는 차별화 기회이다

## 다음 단계 제안

- **점수가 낮은 항목을 보완하려면** → 해당 영역의 전문가/파트너 확보
- **경쟁사 대비 비교가 필요하면** → competitor-mapping 스킬
- **Go 결정 후 제안서를 작성하려면** → proposal-writing 스킬
- **전체 파이프라인을 이어가려면** → `/pipeline`

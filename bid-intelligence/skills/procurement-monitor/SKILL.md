# 조달공고 모니터링

조달공고를 체계적으로 분석하여, 입찰 기회를 신속하게 식별하고 우선순위를 매긴다.

## 이 스킬의 역할

조달공고가 올라오면 짧은 시간 안에 Go/No-Go를 판단해야 한다. 공고 전문을 처음부터 끝까지 읽는 대신, 핵심 정보를 구조화하여 의사결정 속도를 높인다.

## 컨텍스트 활용

이 스킬을 실행하기 전에, 프로젝트의 `_context/` 폴더가 있는지 확인한다.

**폴더가 있으면:**
1. context-loader로 회사 프로필 구축
2. 각 공고의 자격요건을 회사 데이터와 **자동 대조**
3. 적합도 판단에 회사 실적/역량/인력 데이터를 반영
4. 브리핑 형식에 회사 맞춤 분석 포함

**폴더가 없으면:**
- 일반적인 공고 요약만 수행
- 적합도는 사용자에게 질문하여 판단

## 분석 절차

### 1단계: 공고 핵심 정보 추출

공고 텍스트 또는 URL에서 다음 정보를 추출한다:

| 항목 | 내용 |
|---|---|
| 사업명 | 국문/영문 |
| 발주기관 | 기관명 |
| 공고번호 | 식별번호 |
| 공고일 / 마감일 | 남은 일수 표시 |
| 예산규모 | 총액 |
| 사업기간 | 시작~종료 |
| 대상지역 | 해당 시 |
| 사업 분야 | 교육, ICT, 컨설팅 등 |
| 조달 방식 | 경쟁입찰, 제한경쟁 등 |
| 참가자격 | 핵심 자격요건 요약 |
| 유사실적 요건 | 건수, 금액, 분야 |
| 컨소시엄 | 허용/필수/불가 |

### 2단계: 적합성 평가 (회사 데이터 연동)

5가지 기준으로 신호등 평가:

| 기준 | 🟢 Go | 🟡 검토필요 | 🔴 No-Go |
|---|---|---|---|
| 분야 적합성 | 핵심 역량 분야 | 인접 분야 | 비관련 분야 |
| 자격 충족 | 모든 요건 충족 | 일부 불확실 | 핵심 요건 미달 |
| 예산 규모 | 적정 규모 | 과소/과대 | 수익성 없음 |
| 일정 | 충분한 준비 기간 | 촉박하나 가능 | 물리적 불가 |
| 경쟁 강도 | 낮음 (니치) | 중간 | 극심 (레드오션) |

_context/ 데이터가 있으면, 분야 적합성과 자격 충족을 회사 실적/역량 데이터로 자동 판정한다.

### 3단계: 기회 요약 카드

각 공고를 한 장의 요약 카드로 정리한다:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[사업명]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
발주처: [기관명]        예산: [금액]
마감일: [날짜] (D-XX)   기간: [XX개월]
대상: [지역/국가]       분야: [분야]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
적합성: 🟢🟢🟡🟢🟡  종합: Go / 검토 / No-Go
핵심 포인트: [1줄 요약]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 4단계: 복수 공고 우선순위 매트릭스

여러 공고를 동시에 검토할 때 우선순위를 매긴다:

- **영향도** (예산 규모 × 전략적 중요도): High / Medium / Low
- **수주 가능성** (적합성 + 경쟁강도): High / Medium / Low
- 영향도 × 수주가능성 매트릭스에서 우상단(HH)을 최우선 검토

## 브리핑 출력 형식

`/brief` 커맨드에서 호출될 때 사용하는 형식:

```
# 수주 일일 브리핑 — [날짜]

## 신규 공고
| 순위 | 사업명 | 발주처 | 마감일 | 예산 | 적합도 |
|------|--------|--------|--------|------|--------|
| 1 | [사업명] | [기관] | [날짜] (D-XX) | [금액] | 🟢 |
| 2 | ... | ... | ... | ... | 🟡 |

## 마감 임박 (7일 이내)
[해당 공고 목록]

## 추천 액션
- [사업명] → /bid-analyze 권장
- [사업명] → 추가 정보 확인 필요
```

# 산출물 변환 (Export Deliverables)

분석 보고서, 제안서 초안 등을 HWP(한글) 또는 PPT(파워포인트) 문서로 변환한다.

## 이 스킬의 역할

bid-pilot 스킬들의 분석 결과는 마크다운으로 생성된다. 최종 제출물이나 내부 보고용으로 HWP/PPT가 필요할 때, 이 스킬이 마크다운을 올바른 형식으로 변환하고 MCP 서버를 호출한다.

## 마크다운 작성 규칙

hwpx_writer와 pptx_writer가 올바르게 처리하려면 다음 규칙을 반드시 준수해야 한다:

### 헤딩 구조 (HWP용)

마크다운 헤딩 `#`~`######` 6단계로 작성하면, hwpx_writer가 스타일을 자동 적용한다:

- `#` → 제목 (`# 1.`, `# 1.1` 번호 체계)
- `##` → 기호 없는 일반 본문
- `###` → 본문 항목 (기호 자동)
- `####` → 세부 항목 (기호 자동)
- `#####` → 보충 설명 (기호 자동)
- `######` → 참고/주석 (기호 자동)

**기호를 직접 입력하지 않는다.** 헤딩 레벨에 따라 hwpx_writer가 자동 적용한다.

### 금지 사항

- `**볼드**` 마크다운 사용 금지 — 강조는 색상 마커 또는 헤딩 레벨로
- `-` 마크다운 리스트 사용 금지 — 헤딩으로 대체
- 기호 직접 입력 금지 (□, ○, ● 등)

### 색상 마커 (인라인)

- `{{red:텍스트}}` → 로데이터/수치 기반 텍스트 (빨강)
- `{{green:텍스트}}` → 참조문서/RFP 기반 텍스트 (초록)
- 마커 없음 → AI 생성 텍스트 (기본 검정)

### 표

마크다운 테이블 문법 사용:
```
| 구분 | 내용 |
|---|---|
| 항목1 | 설명1 |
```

## 변환 절차

### HWP 변환 (상세 문서용)

1. 분석 결과 마크다운을 위 규칙에 맞게 정제
2. `proposal-body.md` 파일로 저장
3. hwpx_writer MCP의 `convert_text_to_hwpx` 또는 `convert_md_to_hwpx` 도구 호출
4. 생성된 HWPX 파일을 `_output/` 폴더에 저장

### PPT 변환 (프레젠테이션용)

1. 분석 결과에서 핵심 내용을 슬라이드용으로 재구성
2. pptx_writer MCP 도구 순차 호출:
   - `create_pptx` → 세션 생성
   - `add_slides` → 슬라이드 추가 (반복)
   - `finalize_pptx` → 파일 완성
3. 생성된 PPTX 파일을 `_output/` 폴더에 저장

### 변환 유형별 가이드

| 원본 | HWP | PPT |
|------|-----|-----|
| RFP 분석 보고서 | 전체 분석 내용 → 상세 보고서 | 핵심 요약 + 매트릭스 → 브리핑용 |
| 경쟁사 분석 | 프로파일 + 매트릭스 → 내부 참고용 | 매트릭스 + 전략 → 경영진 보고용 |
| 수주 종합 보고서 | 전체 내용 → 팀 공유용 | Executive Summary → 의사결정용 |
| 제안서 초안 | 전체 제안서 → 제출용 문서 | 해당 없음 (제안서는 HWP) |
| 일일 브리핑 | 브리핑 전문 → 이메일/공유용 | 핵심 공고 + 추천 → 주간회의용 |

## 출력 경로

```
프로젝트/
  _output/
    [사업명]-분석보고서.hwpx
    [사업명]-브리핑.pptx
    proposal-body.hwpx
```

- **다른 형식으로도 변환하려면** → HWP↔PPT 전환 가능
- **처음부터 다시 하려면** → `/go`
